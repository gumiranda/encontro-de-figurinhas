## 4. Análise de Concorrentes

### Concorrente 1: TrocaCopa 2026
- **URL:** trocacopa2026.com.br (provável)
- **Pontos Fortes:**
  - Nome de domínio com keyword exata
  - Foco específico na Copa 2026
- **Fraquezas:**
  - Vida útil limitada (só Copa 2026)
  - Sem conteúdo educacional
  - Sem Local SEO

### Concorrente 2: TrocaFigurinhas.com
- **URL:** trocafigurinhas.com
- **Pontos Fortes:**
  - Autoridade estabelecida
  - Múltiplos álbuns
- **Fraquezas:**
  - Interface datada
  - Sem foco em encontros presenciais
  - Sem páginas de cidade otimizadas

### Concorrente 3: ExME App
- **URL:** exme.com.br
- **Pontos Fortes:**
  - App mobile nativo
  - Base de usuários existente
- **Fraquezas:**
  - Não é específico para figurinhas
  - SEO fraco (app-first)
  - Sem conteúdo de blog

### Concorrente 4: Grupos Facebook/WhatsApp
- **Pontos Fortes:**
  - Grande volume de usuários
  - Interação em tempo real
- **Fraquezas:**
  - Não indexável (SEO zero)
  - Desorganizado
  - Difícil encontrar trocas específicas

### Oportunidades vs Concorrência

| Oportunidade | Como Explorar |
|--------------|---------------|
| Local SEO | Páginas de cidade com pontos de troca mapeados |
| Conteúdo educacional | Blog com guias e dicas |
| Long-tail keywords | Páginas para figurinhas específicas |
| Mobile-first | PWA otimizado |
| Comunidade | Perfis de colecionadores, rankings |

## 7. Internal Linking Strategy

### Arquitetura Hub & Spoke

```
                    ┌─────────────────┐
                    │    HOMEPAGE     │
                    │   (Main Hub)    │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
   ┌───────────┐      ┌───────────┐      ┌───────────┐
   │  CIDADES  │      │   COMO    │      │   BLOG    │
   │   (Hub)   │      │ FUNCIONA  │      │   (Hub)   │
   └─────┬─────┘      └───────────┘      └─────┬─────┘
         │                                     │
    ┌────┴────┐                          ┌─────┴─────┐
    │         │                          │           │
    ▼         ▼                          ▼           ▼
┌───────┐ ┌───────┐                  ┌───────┐  ┌───────┐
│ São   │ │ Rio   │                  │ Guia  │  │ Dicas │
│ Paulo │ │ de    │ ...              │ Álbum │  │ Troca │
│       │ │Janeiro│                  │       │  │       │
└───┬───┘ └───────┘                  └───────┘  └───────┘
    │
    ▼
┌──────────────┐
│ PONTOS TROCA │
│   (Spokes)   │
└──────────────┘
```

### Regras de Linking

| De | Para | Anchor Text Pattern |
|----|------|---------------------|
| Homepage | Cidade | "trocar figurinhas em [Cidade]" |
| Homepage | Como Funciona | "veja como funciona" |
| Cidade | Pontos | "[Nome do Ponto]" |
| Cidade | Outras Cidades | "também em [Cidade]" |
| Ponto | Cidade | "mais pontos em [Cidade]" |
| Blog | Cidades | "encontre trocas em [Cidade]" |
| Blog | Como Funciona | "comece a trocar" |
| Qualquer | Homepage | "Figurinha Fácil" (logo/brand) |

### Links por Página (Mínimo)

| Página | Internal Links Mínimo | Para Onde |
|--------|----------------------|-----------|
| Homepage | 10+ | Top cidades, como funciona, blog posts |
| Cidade | 5-8 | Pontos de troca, cidades vizinhas, homepage |
| Ponto | 3-5 | Cidade pai, outros pontos, homepage |
| Blog Post | 5-8 | Páginas relacionadas, CTAs para cadastro |
| Como Funciona | 5+ | Homepage, cidades, blog |

### Anchor Text Guidelines

1. **Evitar:** "clique aqui", "saiba mais", "leia mais"
2. **Preferir:** Keywords descritivas e naturais
3. **Variar:** Não usar sempre o mesmo anchor para a mesma página
4. **Contextual:** Links devem fazer sentido no contexto

### Exemplo de Linking na Prática

```jsx
// Em /cidade/sao-paulo
<p>
  São Paulo tem mais de 50 <Link href="/ponto/shopping-patio-paulista">pontos de troca ativos</Link>, 
  incluindo o famoso encontro no <Link href="/ponto/parque-ibirapuera">Parque Ibirapuera</Link>.
</p>
<p>
  Não encontrou sua figurinha? Veja também colecionadores em 
  <Link href="/cidade/campinas">Campinas</Link> e 
  <Link href="/cidade/santos">Santos</Link>.
</p>
```

### Atualização de Links (Processo)

Quando criar nova página:
1. Identificar 3-5 páginas existentes que devem linkar para ela
2. Adicionar links nessas páginas
3. Na nova página, linkar para 3-5 páginas relevantes existentes
